# Issue #46: Agents must have zero knowledge of GitHub — local-file abstraction

Agents should work with local files only. The runtime translates between local state and the backing platform (GitHub, GitLab, etc.).

Current intent to preserve:
- Issues and epics are treated as local files/work items for agents.
- Orchestrator prompts to agents must use platform-neutral language.
- Adapter/runtime layers own backend-specific translation.

**Wave:** 1  
**Dependencies:** none

## Objective
Ensure all agent-facing orchestration context is platform-neutral so agents operate only on local work items, while the runtime and adapters continue to own all backend-specific identifiers, URLs, and operations.

## Architectural Context
- Trust boundary: agents are untrusted and must not have direct platform knowledge or API access (Constitution Rule 4); runtime mediates side effects via file contracts (Rule 5).
- Agent-facing prompt/artifact assembly currently happens in:
  - `aloop/cli/src/commands/process-requests.ts` (`buildQueuePrompt`, queue prompt rendering)
  - `aloop/cli/src/commands/orchestrate.ts` (estimate/decomposition context blocks, child bootstrap files like `TODO.md` and `TASK_SPEC.md`)
- Platform translation belongs in runtime adapter surfaces (`aloop/cli/src/lib/adapter.ts` and runtime request processors), not in agent prompts.
- This issue introduces a strict separation: backend IDs may remain in machine fields/state, but natural-language prompt text and agent work artifacts must use local "work item" terminology only.

## Scope
In scope for modification:
- `aloop/cli/src/commands/process-requests.ts`
- `aloop/cli/src/commands/orchestrate.ts`
- `aloop/templates/PROMPT_orch_*.md` only where wording is agent-facing and currently platform-specific
- `aloop/cli/src/commands/process-requests.test.ts`
- `aloop/cli/src/commands/orchestrate.test.ts`
- `aloop/cli/src/lib/issue-metadata.ts` and `aloop/cli/src/lib/issue-metadata.test.ts` only if required to remove platform leakage from agent-visible text paths
- Optional new helper(s) under `aloop/cli/src/lib/` for centralized work-item prompt formatting/sanitization

## Out of Scope
Do not modify in this issue:
- `aloop/bin/loop.sh`, `aloop/bin/loop.ps1` (Constitution Rule 1: loop runners stay dumb)
- Direct GitHub command implementation surfaces such as `aloop/cli/src/commands/gh.ts` and `aloop/cli/src/lib/github-monitor.ts` (Rule 12: one concern; this issue is agent-facing abstraction, not backend operation rewrites)
- Dashboard/UI modules under `aloop/cli/dashboard/**` (Rule 19: no gold-plating)
- Full multi-backend feature delivery (e.g., complete `LocalAdapter`, `GitLabAdapter` behavior) beyond what is needed to remove agent-facing platform leakage (Rules 12 and 19)

## Constraints
- Preserve runtime trust-boundary behavior: agents continue writing outputs to `.aloop/output/*` and runtime continues executing external side effects (Constitution Rules 4 and 5).
- Keep the orchestrator event pipeline data-driven; do not hardcode new phase/agent routing in code (Rule 6).
- Maintain existing state machine semantics (`Needs refinement` -> `Ready`/`Blocked`) and existing queue/result processing contracts unless explicitly required for this abstraction.
- Platform identifiers may exist in machine-only fields (state JSON, frontmatter metadata, file naming contracts) but must not appear in agent-facing narrative instructions.
- Add regression tests for every changed behavior (Rule 11).

## Acceptance Criteria
- [ ] Agent-facing queue prompts generated for refinement/estimation use platform-neutral wording (`work item` terminology) and contain no `GitHub`, `PR`, `pull request`, `/issues/`, `/pull/`, or `Issue #<number>` tokens in natural-language sections.
- [ ] Child bootstrap artifacts generated by orchestrator (`TODO.md`, `TASK_SPEC.md`) use platform-neutral wording and do not include platform-specific references.
- [ ] Any required formatting/sanitization logic is centralized (single helper or equivalent) and reused by both `process-requests` and `orchestrate` prompt paths.
- [ ] Existing backend behavior remains intact: no new direct platform calls are introduced in agent-facing prompt assembly code paths.
- [ ] Tests are updated/added in `orchestrate.test.ts` and/or `process-requests.test.ts` to assert the neutral wording and forbidden-token checks.
- [ ] `npm --prefix aloop/cli test` passes.
