# Steering — Activity Log Enhancements

Priority: high (UX regression from previous session)

## Issues

1. **Commit details showing in log — good!** But file changes need more info:
   - Show change type per file: `M` (modified), `A` (added), `D` (deleted), `R` (renamed)
   - Already showing `+69 −2` diffstat — keep that

2. **Missing provider/model info per iteration** — previous dashboard showed which provider AND which model was used for each iteration. Now only shows provider name (claude, copilot). Add the model name too (e.g., "claude · claude-sonnet-4-6" or "opencode · glm-5").

3. **Missing session duration** — no indication of how long the session has been running overall. Add elapsed time since session_start to the header area.

4. **Missing iteration timing context** — individual iterations show duration (e.g., "5m 40s") which is good. But there's no cumulative context — how many iterations total, average time per iteration.

6. **Docs tabs show non-existent files** — the tab bar shows TODO, SPEC, QA LOG, QA COVERAGE, REVIEW LOG, SPEC COMPLIANCE LOG but most of these files don't actually exist in the worktree. Fix:
   - Only show tabs for files that actually exist (check the docs object from the server — if content is empty/null, don't render the tab)
   - If there are many docs, overflow them into a `...` dropdown/ellipsis menu at the end of the tab bar instead of showing a long scrolling row of tabs
