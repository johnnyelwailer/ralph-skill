# Sub-Spec: Issue #91 — QA Coverage Display: Parse and render QA_COVERAGE.md percentage in dashboard

## Objective

Display QA coverage percentage in the dashboard, parsed from `QA_COVERAGE.md` artifact.

## Inputs
- `QA_COVERAGE.md` file generated by QA/review phases
- Existing dashboard session detail area
- `marked` library for markdown rendering (already installed)

## Deliverables
- Backend: `GET /api/qa-coverage` endpoint that reads and parses `QA_COVERAGE.md`
  - Extracts overall coverage percentage (e.g., from "Coverage: 85%" line)
  - Returns `{ percentage: number, raw: string, available: boolean }`
- Frontend: `QACoverageBadge` component
  - Displays coverage percentage as a badge/pill in session detail or header
  - Color-coded: green (>=80%), yellow (60-79%), red (<60%)
  - Click to expand full `QA_COVERAGE.md` content rendered as markdown
- Graceful handling when `QA_COVERAGE.md` doesn't exist (badge hidden or shows "N/A")

## Acceptance Criteria
- [ ] QA coverage percentage displayed from `QA_COVERAGE.md`
- [ ] Coverage badge is color-coded by percentage
- [ ] Click expands to full coverage report
- [ ] Missing `QA_COVERAGE.md` handled gracefully (no error)

## Technical Notes
- Parse percentage with regex: `/Coverage:\s*(\d+)%/i` or similar pattern
- Use `marked` to render the full markdown content in expanded view
- Poll or refresh on SSE `review_complete` events (coverage may update after reviews)
- Keep this component small and self-contained (~50-80 LOC total)
